================================================================================
FORMAT DESCRIPTIONS
================================================================================

MARKDOWN (.md)
  - Human-readable source format
  - Preserves formatting and structure
  - Renders on GitHub, GitLab, many platforms
  - Editable in any text editor
  - Supports bold, italic, headers, links, code blocks
  - Best for: Content management, documentation sites

PLAIN TEXT (.txt)
  - Universal compatibility - works everywhere
  - Monospaced font for readability
  - No special formatting characters
  - Completely archival-safe (will work in 50+ years)
  - Works in any text editor or terminal
  - Best for: Long-term archival, cross-platform use

HTML (.html)
  - Single self-contained file
  - Renders perfectly in any web browser
  - Includes embedded CSS styling
  - Monospaced font for easy reading
  - No external dependencies
  - Best for: Web viewing, email sharing, portability

SPKML (.spkml)
  - Semantic XML format
  - Structured document hierarchy
  - Machine-readable and parseable
  - Preserves logical document structure
  - Based on SPKML concept from CodeBase
  - Best for: Semantic processing, automated parsing, knowledge extraction

================================================================================
READING RECOMMENDATIONS
================================================================================

For quick reading:
  → Use .html files in a web browser

For archival/long-term storage:
  → Use .txt files (universal compatibility forever)

For git/version control repositories:
  → Use .md files (renders automatically on GitHub/GitLab)

For semantic processing/automation:
  → Use .spkml files (structured data)

For editing/modifying:
  → Use .md or .txt files (plaintext, easy to edit)
