SIZE: ~6 KB

PURPOSE: Validates SPKML documents using traditional DTD mechanism

    * URI format validation

  - Cardinality and occurrence rules
  - Namespace support

TRANSFORMATION FILES
================================================================================

FILE: spkml-to-html.xslt
TYPE: XSLT Stylesheet (XSL Transformations v1.0)
SIZE: ~15 KB
FORMAT: Valid XML XSLT v1.0
PURPOSE: Transform SPKML documents to valid HTML5
REQUIREMENTS:
  - libxslt (xsltproc command-line tool)
  - Installation: sudo apt-get install xsltproc (Linux)
                  brew install libxslt (macOS)
USAGE:
  xsltproc spkml-to-html.xslt input.spkml > output.html
  xsltproc --stringparam stylesheet spkml-default.css \
           spkml-to-html.xslt input.spkml > output.html

OUTPUT FEATURES:
  - Valid HTML5 semantic markup
  - Responsive layout (mobile-first)
  - Accessibility features (alt text, semantic elements)
  - Professional styling with included CSS
  - Auto-generated table of contents (optional)
  - Auto-numbered figures and tables
  - Proper heading hierarchy preservation
  - Media embedding (audio, video)

TRANSFORMATIONS INCLUDED:
  - All structural elements → HTML sections
  - Headings → <h1> through <h6>
  - Paragraphs → <p> with inline formatting
  - Lists → <ul> and <ol>
  - Code blocks → <pre><code>
  - Figures → <figure><img><figcaption>
  - Media → <audio>, <video> with controls
  - Tables → <table><thead><tr><td>
  - Quotes → <blockquote>
  - References → <a> with metadata
  - Concepts → Styled <section> with semantic classes

FILE: spkml-default.css
TYPE: Cascading Style Sheets (CSS3)
SIZE: ~11 KB
FORMAT: Valid CSS3 with Media Queries
PURPOSE: Professional, accessible styling for SPKML HTML output
FEATURES:
  - Responsive design for mobile, tablet, desktop
  - Dark mode support via prefers-color-scheme
  - Print-friendly styles
  - Accessibility features:
    * High contrast mode support
    * Reduced motion support
    * Focus visible for keyboard navigation
    * Proper link styling for understandability
  - Professional typography
  - Color scheme variables (customizable)
  - Syntax highlighting for code blocks

RESPONSIVE BREAKPOINTS:
  - Desktop: max-width 900px centered
  - Tablet: adjustments for max-width 768px
  - Mobile: optimizations for max-width 480px
  - Print: optimized for paper output

CUSTOMIZATION:
  All colors/fonts defined as CSS variables at :root:
  - --primary-color: #333333
  - --secondary-color: #666666
  - --accent-color: #0066cc
  - --font-family-sans: System fonts
  - --font-family-mono: Courier New

USAGE:
  1. Include in HTML: <link rel="stylesheet" href="spkml-default.css">
  2. Or customize and copy styles inline
  3. Or extend with additional stylesheets

================================================================================
QUICK REFERENCE: WHICH FILE TO USE WHEN
================================================================================

IF YOU WANT TO...

1. GET STARTED QUICKLY
   → Read: SPKML_README.md (Quick Start section)
   → Use: SPKML_Template_Examples.md (pick appropriate template)
   → Validate: spkml.xsd

2. UNDERSTAND SPKML COMPLETELY
   → Read: SPKML_Specification_1.0.md (start to finish)
   → Reference: spkml.xsd (for detailed element rules)
   → Copy: SPKML_Template_Examples.md (for real examples)

3. CREATE A NEW DOCUMENT
   → Copy: Appropriate template from SPKML_Template_Examples.md
   → Edit: Your content
   → Validate: xmllint --schema spkml.xsd your-document.spkml
   → Transform: xsltproc spkml-to-html.xslt your-document.spkml > output.html

4. VALIDATE YOUR DOCUMENT
   → Use: spkml.xsd (modern, recommended)
   → Or: spkml.dtd (legacy)
   → Tool: xmllint, VS Code, Oxygen XML Editor

5. TRANSFORM TO HTML
   → Use: spkml-to-html.xslt
   → Then: Apply spkml-default.css to output
   → Tool: xsltproc (command line)

6. STYLE YOUR HTML OUTPUT
   → Use: spkml-default.css
   → Or: Customize CSS variables in spkml-default.css
   → Or: Write additional CSS to override defaults

7. PUBLISH TO DIFFERENT FORMATS
   → HTML: Use spkml-to-html.xslt + spkml-default.css
   → PDF: Transform to HTML, then use wkhtmltopdf or Prince
   → EPUB: Transform to HTML, package as EPUB container
   → Plain Text: Write custom XSLT transformer
   → Markdown: Write custom XSLT transformer

SUMMARY
================================================================================

SPKML provides a complete ecosystem for semantic document creation:

1. DEFINE: SPKML_Specification_1.0.md defines the language
2. VALIDATE: spkml.xsd and spkml.dtd ensure correctness  
3. CREATE: SPKML_Template_Examples.md provides starting points
4. TRANSFORM: spkml-to-html.xslt converts to HTML
5. STYLE: spkml-default.css provides professional appearance
6. DEPLOY: Output HTML to web, convert to PDF, etc.

All files work together to provide a complete solution for semantic,
format-agnostic document creation and publishing.
